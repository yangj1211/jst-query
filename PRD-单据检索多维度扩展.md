# 产品需求文档（PRD）：单据检索多维度扩展

## 1. 产品概述

### 1.1 背景

原有单据检索模块仅支持"销售订单"单一维度，用户只能以销售订单为入口检索关联文件。随着业务发展，用户需要从采购单、独立凭证等多个维度检索单据，以满足财务审计、供应链管理、合规检查等场景需求。

### 1.2 目标

将单据检索模块从单一"销售订单"维度扩展为三个维度：**销售订单**、**采购单**、**凭证**。用户在同一检索入口输入问题，系统同时在三个维度中检索，右侧结果面板按维度分 Tab 展示，每个维度独立筛选、分页、批量操作。

### 1.3 用户价值

- 一次提问，三个维度同时检索，减少重复操作
- 按维度分组展示，结果清晰有层次
- 凭证维度支持结构化字段检索，满足审计场景
- 文件预览弹窗，快速查看文件内容

---

## 2. 功能范围

### 2.1 三个检索维度

| 维度 | 主体 | 说明 |
|------|------|------|
| 销售订单 | 以销售订单号为主键，关联该订单下的所有文件 | 保留原有功能，扩展文件类型 |
| 采购单 | 以采购订单号为主键，关联该采购单下的所有文件 | 新增维度 |
| 凭证 | 以文件本身为主体，不关联订单，每类凭证有独立的结构化字段 | 新增维度 |

### 2.2 不在范围内

- 真实文件存储和上传
- 权限管理后台配置
- 与外部系统的数据对接

## 4. 功能详细描述

### 4.1 统一检索入口

- 用户在左侧对话框输入自然语言问题
- 系统解析查询条件，同时在三个维度执行检索
- 支持的查询条件：客户/供应商名称、时间范围、金额范围、文件类型、订单号、行业、地区等
- 支持通用关键词全文搜索（如"上海电气的相关单据"）
- 查询中包含文件类型关键词时，自动映射为各维度的筛选条件

### 4.2 对话区结果展示

- 摘要信息：显示各维度命中数量（如"共检索到销售订单 5 条，采购单 3 条，凭证 8 份"）
- 结果分三个区块展示：销售订单、采购单、凭证
- 每个区块显示表格（默认每页20条），有独立分页
- 每个区块右上角有"查看"按钮，点击自动切换右侧面板到对应维度 Tab
- 某维度无结果时不显示该区块

### 4.3 思考过程展示

检索过程分4个步骤动画展示：
1. 理解问题 — 识别查询意图
2. 在销售单据中检索
3. 在采购单据中检索
4. 在独立单据中检索

### 4.4 右侧结果面板（多维度 Tabs）

- 顶部使用 Tabs 组件，三个页签：**销售订单**、**采购单**、**凭证**
- 每个页签标签显示该维度命中数量（如"销售订单 (22)"）
- 检索完成后默认激活第一个有结果的页签
- 各页签状态独立（筛选、分页、选中、展开互不影响）

### 4.5 销售订单 Tab

- 卡片式展示：订单号、项目名称、客户名称、凭证日期、合同金额
- 展开后显示该订单关联的文件列表（受筛选条件过滤）
- 点击详情按钮打开订单详情抽屉（基础信息、发票信息、付款信息、物料清单、关联单号）
- 独立的文件类型筛选器（有权限/无权限分组，"有权限"绿色标签）
- 独立的分页、全选、批量下载

### 4.6 采购单 Tab

- 卡片式展示：采购订单号、供应商名称、采购日期、采购金额
- 展开后显示该采购单关联的文件列表（受筛选条件过滤）
- 点击详情按钮打开采购单详情抽屉（仅基础信息，不含文件列表）
- 采购单详情字段：采购订单号、供应商名称、采购日期、采购金额、采购类型、关联销售订单、状态、描述
- 独立的文件类型筛选器（有权限/无权限分组）
- 独立的分页、全选、批量下载

### 4.7 凭证 Tab

- 卡片式展示：文件名（主字段）、凭证类型标签（彩色区分）、预览按钮
- 每条记录下方展示该类型对应的结构化字段（根据 docCategoryMeta 动态渲染）
- 27种凭证类型各有独立的彩色标签，颜色差异明显
- 点击预览按钮弹出居中 Modal 弹窗，展示文件预览（PDF 模拟阅读器样式、图片预览等）
- 独立的凭证类型筛选器
- 独立的分页、全选、批量下载

### 4.8 文件预览

- 以居中 Modal 弹窗形式展示（非右侧抽屉）
- PDF 文件：深色背景 + 白色文档页面模拟
- 图片文件：居中展示图片预览
- 其他格式：显示文件图标和格式信息
- 弹窗宽度 70%，高度 75vh

### 4.9 文件类型筛选器

- 下拉多选，支持"全部"选项
- 分组显示：**有权限**（绿色标签）和 **无权限**（灰色标签）
- 各维度的有权限类型独立配置
- 选择"全部"时清空具体类型选择；选择具体类型时移除"全部"
- 切换维度 Tab 时保留各维度已选择的筛选状态
- 搜索问题中包含文件类型关键词时，自动设置对应筛选条件

## 5. 检索逻辑

### 5.1 查询解析

- 支持自然语言解析：客户名称、订单号、时间范围、金额条件、文件类型、行业、地区
- 支持聚合查询（按客户/地区/行业统计）
- 兜底逻辑：无法解析出结构化条件时，自动提取关键词（去除"的"、"相关"、"单据"等停用词），进行全文搜索

### 5.2 销售订单维度检索

- 按订单字段（客户、日期、金额等）过滤订单
- 按文件类型过滤关联文件
- 全文关键词匹配订单所有字段 + 关联文件名和标签

### 5.3 采购单维度检索

- 销售维度条件自动映射为采购字段（customer→supplier, poDate→purchaseDate, amount→purchaseAmount, orderNo→purchaseOrderNo, title→description）
- 按文件类型过滤关联文件
- 全文关键词匹配采购单所有字段 + 关联文件名和标签

### 5.4 凭证维度检索

- 按凭证类型（docCategory）过滤
- 全文关键词匹配：文件名 + 凭证类型 + 结构化字段所有值
- 支持金额条件匹配结构化字段中的数值
- 支持时间条件匹配结构化字段中的日期

## 6. 数据模型

### 6.1 采购订单主表

| 字段 | 类型 | 说明 |
|------|------|------|
| purchaseOrderNo | string | 采购订单号（主键） |
| supplier | string | 供应商名称 |
| purchaseDate | string | 采购日期 |
| purchaseAmount | string | 采购金额 |
| currency | string | 币种 |
| purchaseType | string | 采购类型（设备采购/原材料采购/零部件采购） |
| relatedSalesOrder | string | 关联销售订单号 |
| status | string | 状态（已完成/执行中/待发货） |
| description | string | 描述 |

### 6.2 采购单据关联表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 文件 ID（主键） |
| purchaseOrderNo | string | 采购订单号（外键） |
| type | string | 文件格式（pdf/image） |
| name | string | 文件名称 |
| tag | string | 文件标签（15种类型之一） |
| tagColor | string | 标签颜色 |
| path | string | 文件路径 |

### 6.3 独立凭证表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 文件 ID（主键） |
| fileName | string | 文件名称 |
| docCategory | string | 凭证类型（27种之一） |
| fileFormat | string | 文件格式（pdf/jpg/xlsx/docx） |
| structuredFields | object | 该类凭证特有的结构化字段值 |

### 6.4 凭证类型元数据

为每种凭证类型定义结构化字段的元数据配置（字段名、中文标签、字段类型），用于动态渲染凭证卡片的结构化信息。

## 7. 交互规范

| 交互场景 | 行为 |
|----------|------|
| 输入问题并回车 | 4步思考动画 → 三维度并行检索 → 分区块展示结果 |
| 点击对话区"查看"按钮 | 右侧面板切换到对应维度 Tab，重置该维度筛选状态 |
| 切换右侧 Tab | 保留各维度独立的筛选、分页、选中状态 |
| 选择文件类型筛选 | 仅过滤当前维度的结果，不影响其他维度 |
| 点击订单/采购单详情 | 打开右侧抽屉展示详情 |
| 点击凭证预览 | 弹出居中 Modal 展示文件预览 |
| 搜索含文件类型关键词 | 自动设置三个维度的筛选条件为该文件类型 |
| 折叠/展开右侧面板 | 浮动按钮控制，不影响对话区 |
